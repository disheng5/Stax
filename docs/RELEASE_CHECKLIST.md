# Stax MVP 发布 Checklist

> 跟着这份清单一步步执行即可把 MVP 提交微信审核。预计耗时 1-2 小时。

---

## 0. 前置准备

- [ ] 已注册微信小程序 AppID（个人主体即可）
- [ ] 微信开发者工具最新版
- [ ] 云开发环境已开通，记下 **环境 ID**

---

## 1. 替换 AppID 与环境 ID

1. **AppID**
   - 编辑 `project.config.json` → `"appid"` 改为你的 AppID
2. **云环境 ID**
   - 编辑 `miniprogram/app.js` → 把 `const ENV_ID = 'cloud1-xxx'` 改为你的环境 ID
3. 在开发者工具里点 `详情` → 关闭「不校验合法域名」（提审需关闭）

---

## 2. 云资源准备

### 2.1 建库（5 个集合）

按 `docs/DEPLOY.md §3.1` 在「云开发 → 数据库」逐个新建：

```
users
games
transactions
terms
handRanks
```

### 2.2 配置权限

按 `docs/DEPLOY.md §3.2` 配置每个集合的数据权限。

### 2.3 部署 19 个云函数

对 `cloudfunctions/` 下每个目录右键 → **上传并部署：云端安装依赖**：

```
whoami
createGame
joinGame
recordTransaction
settleGame
seedTerms
aiReview
termAi
getAvatars
createCircle
joinCircle
leaveCircle
dissolveCircle
calcSeasonScore
settleSeason
resetSeason
excludeGame
deleteGameRecord
removeCircleMember
```

### 2.4 灌入种子数据

在「云开发 → 云函数 → seedTerms → 云端测试」运行：

```json
{ "reset": true, "only": "terms" }
```

返回 ok 后再运行：

```json
{ "reset": true, "only": "handRanks" }
```

预期：术语 50 条，起手牌 169 条。

## 3. 本地自测

### 3.1 单元测试

```bash
npm install
npm test
```

预期：所有 ✓。

### 3.2 关键路径手测

在开发者工具里至少跑一遍：

- [ ] 首页 hero 展示 Stax / 长河筹略 / 一句兵法
- [ ] 点 `开局` 创建牌局，默认 5/5/500，权限共享开启
- [ ] 分享给另一个微信号点击链接，能直接进入牌局并看到“上桌”按钮
- [ ] 玩家自己卡片 `补码 2 手`，手数 / 总码量正确累计
- [ ] 玩家自己卡片点 `下桌`，填手头筹码，状态变 `已下桌`
- [ ] 房主点 `最终结算`，提交后自动出结算图
- [ ] 结算图含战报 + 一句兵法
- [ ] AI 点评出现且有兵法引用
- [ ] 战绩页可删除自己的记录，删除后列表不再出现
- [ ] 围观者切换上桌正常

## 5. 提交审核

1. 开发者工具 → 右上角 `上传`
2. 版本号填 `0.1.0`，备注 `Stax MVP`
3. 登录 [mp.weixin.qq.com](https://mp.weixin.qq.com) → `版本管理` → `开发版本` → `提交审核`

### 5.1 必填项

| 项       | 推荐内容                                                                            |
| -------- | ----------------------------------------------------------------------------------- |
| 类目     | 工具 → 实用工具                                                                     |
| 名称     | Stax · 长河筹略                                                                     |
| 简介     | 朋友局德州扑克筹码记账、盲注计时、AA 分摊与战绩统计工具，附扑克术语词典与 AI 复盘。 |
| 功能截图 | 5 张：首页 / 牌局详情 / 下桌弹窗 / 最终结算 / 战绩页                                |
| 测试账号 | 可选填两个微信号；说明 _"用账号 A 点开局生成邀请码，账号 B 点击分享链接上桌"_       |
| 隐私协议 | 见 `docs/PRIVACY.md`                                                                |

### 5.2 审核可能驳回点

- 注意：扑克类小程序容易被审核要求强化合规说明
- 在「关于」页和首次启动弹窗里明确：
  - **仅供朋友间线下竞技记账与学习交流**
  - **不提供任何形式的真实金钱交易**
  - **严禁用于赌博活动**
- 如审核反馈具体问题，按反馈回复并补充截图

---

## 6. 审核通过后

1. 进入 `版本管理 → 审核版本` 点 `发布`
2. 小程序立即上线
3. 通过微信搜索 `Stax` 或扫码即可使用

## 7. 上线后建议

- 第一周收集 3-5 位真实朋友局的反馈
- 重点关注：分享上桌路径、下桌交互、结算图体验
- 后续可考虑接入真模型（混元 / 云开发 AI）替换 `aiReview` 模板

## 附：未认证小程序的分享限制

微信平台规则：**个人主体未认证的小程序，分享按钮会弹「小程序未完成认证，分享功能暂时无法使用」**。

这是平台行为，不是代码问题。

解决方案：

1. **提审通过后**，分享对一般微信用户是放开的（少数版本仍受限）。
2. **走小程序认证**（个人主体 30 元 / 年）：
   - 登录 mp.weixin.qq.com → `设置 → 基本设置 → 微信认证`
   - 认证通过后分享、模板消息等能力全部开放。
3. 在认证前，可让朋友通过：
   - 复制邀请码手动加入（开局页/详情页都能复制）
   - 你扫别人的微信扫一扫，对方扫你二维码进入
