# Stax 演示脚本（DEMO）

> 给第一次跑 Stax 的你 / 你的朋友。**全程 8 分钟**，能把 v0.2 所有核心能力跑一遍：创建 → 自助补码 → 撤销 → 淘汰 → AA 结算 → AI 点评 → 数据看板。

## 🧪 两种演示模式

| 模式 | 何时用 | envId | 数据 |
|---|---|---|---|
| **Demo 模式（默认）** | 还没开通云开发 / 想快速看 UI | 不用配 | 内存 mock，刷新即重置 |
| **真实云开发模式** | 已开通环境、要做真机回归 | 必须配 | 持久化、可多人协作 |

### Demo 模式快速体验（推荐你今晚先跑这个）

1. 打开微信开发者工具 → 导入项目（AppID 用测试号即可）
2. 直接点「编译」**不需要任何配置**
3. 控制台会看到 `[cloud-mock] installed — running in DEMO MODE`
4. 进首页就能看到：
   - 1 个进行中的「今晚的 Demo 局」（你 + Bob + Carol）
   - 「我的」→ 显示已有 5 局历史 + 累计盈亏 +230
   - 「数据看板」→ 趋势图 + 4 维度都有真实数据
5. 「我的」→「🧪 重置 Demo 数据」可恢复初始状态

⚠️ **Demo 模式限制**：
- 数据仅存在当前小程序运行时内存中
- 模拟"你 + Bob + Carol"已经互相打过 5 局
- 无法多设备协作（毕竟没真实云）
- 启动时会显示金色「🧪 Demo 模式」横幅

切换到真实模式：编辑 `miniprogram/app.js` 把 `ENV_ID` 改为真实环境 ID。

> ⚠️ 真实云开发模式需先完成 `docs/DEPLOY.md` 三步（envId、6 个集合、6 个云函数 + `seedTerms`）。

| **玩家 Bob**   | 第二个微信号    | 真机扫码加入即可 |

如果只有 1 部手机：在开发者工具切「**测试号**」编译两次，模拟 Bob 和 Carol。

## 步骤

### Step 1. Alice 完善资料（30s）

1. 打开小程序 → 看到首页「使用须知」弹窗 → 点「我已知晓」
2. 底 tab → 「我的」
3. 点头像 → 选一张图 → 输入昵称「Alice」→ 点「保存资料」

**预期**：头像/昵称写入云端，stat 区显示 `0 场次 / 0 累计盈亏 / 0% 胜率`。

---

### Step 2. Alice 创建一局（1min）

1. 底 tab → 「首页」→ 点「**创建牌局**」大按钮
2. 局名保留默认（今天日期 + "牌局"）
3. 标准买入额：`100`
4. 盲注结构：选 「**标准 10/20**」
5. 升盲间隔：`20` 分钟
6. 点「生成牌局并获取邀请码」

**预期**：弹出邀请码（如 `K7F3PX`），点确定 → 进入牌局详情页。

---

### Step 3. Bob & Carol 加入（1min）

**方式 A（推荐）**：Alice 在详情页点「**分享邀请**」→ 微信选 Bob/Carol → 他们点开小程序自动跳到加入页

**方式 B**：把邀请码报给 Bob/Carol → 各自打开小程序 → 「加入牌局」→ 输入 6 位码

**预期**：
- Alice 屏幕**实时**多出 Bob/Carol 两张玩家卡（无需刷新，靠 `db.watch()`）
- 顶部「总池」从 100 变成 300
- 三人卡片都显示 `买入 100 · 次数 1`

---

### Step 4. 演示「自助补码」与「庄家撤销」（2min）

#### 4.1 Bob 自助补码
- Bob 在自己屏幕上找到自己的卡（带绿色「我」标签）→ 点「**+ 我要补码**」
- 弹窗输入 `100` → 确定

**预期**：
- Bob 卡 `买入 200 · 次数 2`
- 总池 `400`
- Alice 屏幕「最近流水」区出现一条 `Rebuy Bob 100 自助`

#### 4.2 Alice 代 Carol 补码
- Alice 在 Carol 的卡上点「代补」→ 输入 `100`

**预期**：
- Carol 卡 `买入 200 · 次数 2`
- 总池 `500`
- 流水多一条 `Rebuy Carol 100`（无"自助"标识，因为庄家代操）

#### 4.3 Alice 撤销 Bob 那次自助
- Alice 在流水那条 `Rebuy Bob 100 自助` 旁点「**撤销**」→ 二次确认

**预期**：
- Bob 卡回到 `买入 100 · 次数 1`
- 总池回到 `400`
- 流水那条变灰 + 显示「已撤销」

> 💡 这一步演示了**信任 + 兜底**机制：参与人能自助记录降低庄家负担；如果记错了庄家可以一键回滚，不需要扯皮。

---

### Step 5. Alice 淘汰 Bob（30s）

- Alice 在 Bob 卡上点「淘汰」→ 确认

**预期**：Bob 卡变灰，stack 显示「已淘汰」，三人屏幕同步刷新。

---

### Step 6. Alice 结束牌局并 AA 结算（2min）

- Alice 点「结束并结算」→ 进入结算页

#### 6.1 输入最终筹码
| 玩家 | 买入 | 最终筹码 |
|---|---|---|
| Alice | 100 | **350** |
| Bob   | 100 | **0**（淘汰自动填 0） |
| Carol | 200 | **150** |
| 校验 | | Σ profit = +250 - 100 - 50 = **+100** ❌ 不为 0 |

调整 → 把 Alice 改成 **250**：

| Alice | 100 | 250 | profit = +150 |
| Bob   | 100 | 0   | profit = -100 |
| Carol | 200 | 150 | profit = -50 |
| 校验 | | Σ = **0** ✅ |

#### 6.2 添加额外费用 AA
- 「额外费用」总额：`60`（场地费）
- 模式选「**赢家按比例**」

**预期**：
- 因为只有 Alice 是赢家，AA 全部由 Alice 担：`Alice -60`
- finalProfit：Alice +90 / Bob -100 / Carol -50（合计 -60，等于额外费用）
- 「清算建议」给出 ≤ 2 笔转账，比如 `Bob → Alice 90` + `Carol → Alice 50` 或类似最优解

#### 6.3 提交结算
- 点「提交结算」→ 弹「结算完成」
- 点「**🤖 看看 AI 怎么点评今晚**」

---

### Step 7. AI 复盘（30s）

进入复盘页：
1. 看到大脑 emoji loading 动画
2. 1 秒内出现一段牙贱口风格的 200 字点评，例如：
   > "聊了整整 X 分钟，3 个人围着桌子转了 X 圈，确认过眼神，是有故事的人。今晚的 MVP 是 Alice，独吞 90，运气和算计都拿满分。心态最稳的是 Bob，-100 也能笑着结账..."
3. 下方「本局事实」卡列出 玩家数 / 时长 / 总池 / 补码次数 / 大赢家 / 大输家
4. 三个按钮：「复制点评」「换一段」「分享好友」

**预期**：
- 「换一段」会调用云函数重新生成（规则模板有一定随机性，多人时也会有差异）
- 「分享好友」生成一张分享卡，标题 "Stax AI 给我的牌局点评，看看准不准 😎"

---

### Step 8. 验收数据看板（1min）

- 返回首页 → 「我的」→ 「数据看板」

**预期**：
- 概览区：`总场次 1 / 累计盈亏 +90 / 胜率 100%`（你是 Alice）
- 趋势图：1 个数据点（首局基线）
- 维度分析默认「人数」tab：`3 人 · 1 场 · +90`
- 切「补码次数」tab：`1 次买入 · 1 场 · +90`
- 切「星期」tab：会显示当前是周几
- 切「对手」tab：列出 `Bob` `Carol` 各 1 场（这俩对手目前只跟你打过 1 局）

## 预期问题排查

| 现象 | 原因 | 解决 |
|---|---|---|
| 启动弹「尚未配置云环境」 | `app.js` 里 envId 还是占位符 | 替换为真实 envId |
| 创建牌局点没反应 | 云函数没部署 | 在云开发面板上传 6 个云函数 |
| 加入牌局报「邀请码无效」 | 大小写敏感 / 空格 | 邀请码全大写、无空格 |
| 实时同步不刷新 | DB 权限不对 | `games` 集合权限设「所有人可读，仅创建者可写」 |
| AI 点评空白 / 卡 loading | `aiReview` 云函数没部署 | 部署 `aiReview`，默认走规则模板可立即生效 |
| 看板没数据 | 还没结算过任何牌局 | 先按本脚本走一遍 Step 1~6 |
